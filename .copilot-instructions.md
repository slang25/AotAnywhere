# GitHub Copilot Development Instructions

This document provides development guidance for the PublishAotCross repository, specifically focused on helping with agent tasks and iterating on the clang wrapper scripts.

## Repository Overview

PublishAotCross is a NuGet package that enables cross-compilation for .NET Native AOT applications using Zig as the linker/sysroot. The primary components developers work with are:

- `src/clang` - Bash script wrapper for Zig cc on Unix systems
- `src/clang.cmd` - Batch script wrapper for Zig cc on Windows
- `src/Crosscompile.targets` - MSBuild targets that configure Zig toolchain
- `src/PublishAotCross.targets` - Main MSBuild integration

## Required Development Dependencies

### Zig 0.15.1 Installation

The repository requires **Zig 0.15.1** for development and testing. The MSBuild targets use Vezel.Zig.Toolsets version `0.15.1.1` which corresponds to Zig 0.15.1.

#### Windows Installation

```powershell
# Download Zig 0.15.1 for Windows
$url = "https://ziglang.org/download/0.15.1/zig-windows-x86_64-0.15.1.zip"
$outputPath = "$env:TEMP\zig-windows-x86_64-0.15.1.zip"
Invoke-WebRequest -Uri $url -OutFile $outputPath

# Extract to C:\zig
Expand-Archive -Path $outputPath -DestinationPath "C:\"

# Add to PATH (for current session)
$env:PATH = "C:\zig-windows-x86_64-0.15.1;$env:PATH"

# Permanently add to PATH
[Environment]::SetEnvironmentVariable("PATH", "C:\zig-windows-x86_64-0.15.1;$env:PATH", "User")
```

#### macOS Installation

```bash
# Download Zig 0.15.1 for macOS
cd /tmp
curl -L "https://ziglang.org/download/0.15.1/zig-macos-x86_64-0.15.1.tar.xz" -o zig-macos-x86_64-0.15.1.tar.xz

# For Apple Silicon Macs, use:
# curl -L "https://ziglang.org/download/0.15.1/zig-macos-aarch64-0.15.1.tar.xz" -o zig-macos-aarch64-0.15.1.tar.xz

# Extract and install
tar -xf zig-macos-x86_64-0.15.1.tar.xz
sudo mv zig-macos-x86_64-0.15.1 /usr/local/zig

# Add to PATH
echo 'export PATH="/usr/local/zig:$PATH"' >> ~/.zshrc
source ~/.zshrc
```

#### Linux Installation

```bash
# Download Zig 0.15.1 for Linux x64
cd /tmp
wget "https://ziglang.org/download/0.15.1/zig-linux-x86_64-0.15.1.tar.xz"

# For ARM64 systems, use:
# wget "https://ziglang.org/download/0.15.1/zig-linux-aarch64-0.15.1.tar.xz"

# Extract and install
tar -xf zig-linux-x86_64-0.15.1.tar.xz
sudo mv zig-linux-x86_64-0.15.1 /opt/zig

# Add to PATH
echo 'export PATH="/opt/zig:$PATH"' >> ~/.bashrc
source ~/.bashrc
```

#### Verification

After installation, verify Zig is properly installed:

```bash
zig version
# Should output: 0.15.1
```

## Development Workflow

### Building the Project

```bash
# Build the NuGet package
dotnet pack src/PublishAotCross.nuproj -c Release

# Test with the sample Hello World project
dotnet build test/Hello.csproj

# Test cross-compilation (requires Zig)
dotnet publish test/Hello.csproj -r linux-x64
```

### Testing Clang Scripts

The clang wrapper scripts are the core of cross-compilation functionality. When iterating on these scripts:

#### Testing the Bash Script (`src/clang`)

```bash
# Make the script executable
chmod +x src/clang

# Test basic zig availability check
./src/clang --version

# Test with sample compilation arguments (Linux target)
./src/clang -o test_output -target x86_64-linux-gnu test.c

# Test macOS cross-compilation detection
./src/clang -o test_output -target x86_64-apple-darwin test.c
```

#### Testing the Windows Batch Script (`src/clang.cmd`)

```cmd
# Test basic zig availability check
src\clang.cmd --version

# Test with sample compilation arguments
src\clang.cmd -o test_output.exe -target x86_64-linux-gnu test.c
```

### Common Development Tasks

#### Modifying Cross-Compilation Logic

When modifying the clang wrapper scripts, focus on these key areas:

1. **Target Detection**: Logic that determines if compiling for Linux or macOS
2. **Argument Filtering**: Removing unsupported flags for Zig
3. **Library Handling**: Managing system libraries and frameworks
4. **Path Configuration**: Setting up SDK paths and search directories

#### Testing Changes

```bash
# Test all supported target platforms from repository root
dotnet publish test/Hello.csproj -r linux-x64
dotnet publish test/Hello.csproj -r linux-arm64
dotnet publish test/Hello.csproj -r linux-musl-x64
dotnet publish test/Hello.csproj -r linux-musl-arm64
dotnet publish test/Hello.csproj -r osx-x64     # Experimental
dotnet publish test/Hello.csproj -r osx-arm64   # Experimental
```

#### Debugging Cross-Compilation Issues

1. **Enable Verbose Output**: Add debug echo statements to clang scripts
2. **Check Zig Command**: Review the final `zig cc` command being executed
3. **Validate Arguments**: Ensure unsupported flags are properly filtered
4. **Test Incrementally**: Make small changes and test after each modification

### Using External Zig vs. NuGet Package

The repository supports two modes for Zig usage:

#### Default: NuGet Package (Recommended for CI/CD)
```bash
# Uses Vezel.Zig.Toolsets package version 0.15.1.1
dotnet publish test/Hello.csproj -r linux-x64
```

#### External Zig (Recommended for Development)
```bash
# Uses system-installed Zig from PATH
dotnet publish test/Hello.csproj -r linux-x64 /p:UseExternalZig=true
```

For development and iteration on clang scripts, using external Zig is preferred as it:
- Provides faster iteration cycles
- Allows easier debugging with verbose output
- Enables testing with different Zig versions

## Troubleshooting

### Common Issues

#### "zig is not on the PATH"
- Verify Zig 0.15.1 is installed and in PATH
- Check with `zig version` command
- Restart terminal/IDE after PATH changes

#### Cross-compilation failures
- Check that target platform flags are correctly detected
- Verify system libraries are available
- Review filtered arguments in script output

#### macOS cross-compilation limitations
- Experimental feature with known limitations
- Some .NET libraries requiring Objective-C runtime may fail
- Use native macOS builds for production

### Getting Help

- Review existing clang script logic for similar scenarios
- Check GitHub Actions workflows for CI/CD examples
- Consult Zig documentation for compiler flag compatibility
- Test with minimal reproduction cases

## Code Style Guidelines

When modifying clang wrapper scripts:

1. **Maintain cross-platform compatibility** between bash and batch scripts
2. **Add comprehensive comments** for complex argument filtering logic
3. **Use consistent variable naming** (`is_macos_target`, `is_cross_compiling_to_macos`)
4. **Preserve existing error handling** patterns
5. **Test edge cases** like empty arguments or special characters

## Performance Considerations

- Zig compilation is generally fast, but cross-compilation may take longer
- Consider using external Zig for development to avoid NuGet package download overhead
- Large applications with many dependencies may have longer link times
